## 决策记录

1. **Windows TCP 僵尸条目问题**：netstat 报告的 PID 可能与实际进程不匹配（进程已死亡但 TCP 表未清理）。可通过 `Get-NetTCPConnection` 的 ProcessName 字段（显示 `<DEAD>`）或 `Get-Process` 确认进程是否真实存在。
2. **服务停止策略**：当单个 PID 无法定位到真实进程时，需终止所有相关 Python 进程以确保服务完全停止。僵尸 TCP 条目会随关联进程终止自动清理。
3. **端口可用性验证**：即使 netstat 显示端口被占用，只要对应 PID 已死亡，SO_REUSEADDR 仍可成功绑定端口。
