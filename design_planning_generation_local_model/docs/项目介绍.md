# QMS文档生成系统（project_0430_02_beta）项目介绍

## 1. 项目概述

**QMS文档生成系统**是一个面向医疗器械企业的AI驱动质量体系文档自动生成工具。该系统基于大语言模型（LLM），结合RAG（检索增强生成）技术和Web搜索能力，能够根据用户输入的产品信息和文档类型，自动生成符合ISO 13485、ISO 14971、GB 42062等国际/国家法规标准的专业质量体系文档，输出为Word（.docx）格式。

### 1.1 核心价值

- **降本增效**：将质量体系文档编写从数天缩短至数分钟，大幅降低人工编写成本
- **合规保障**：内置法规标准知识库，自动引用相关标准条款，确保文档合规性
- **专业质量**：生成内容达到注册申报级别的详细程度和专业水平

### 1.2 支持的文档类型（16种）

| 类别 | 文档类型 | 说明 |
|------|----------|------|
| 风险管理 | 风险管理报告 | 符合ISO 14971/GB 42062的综合风险管理报告 |
| 风险管理 | 风险管理计划 | 定义风险管理活动计划和职责分配 |
| 风险管理 | FMEA分析报告 | 失效模式与效应分析 |
| 风险管理 | 风险可接受准则 | 风险判定标准和RPN准则 |

### 3.1 技术栈总览

## 4. 核心代码模块详解

### 4.1 Web前端（app/static/index.html，1014行）

单一HTML文件包含完整的CSS样式和JavaScript逻辑，实现以下功能：

- **自定义组合框**：支持搜索过滤、键盘导航（↑↓Enter Esc）、分组显示（风险管理/设计开发/注册申报/通用）
- **附件上传**：拖拽上传 + 点击选择 + 轮询提取状态，支持.docx/.pdf/.txt
- **异步生成**：提交任务后轮询状态，实时更新进度条
- **搜索日志**：展示每个章节使用的搜索方式（Agent SDK / Playwright / 无）
- **响应式UI**：渐变背景、卡片式布局、加载动画

### 4.2 API路由（app/api/routes.py，255行）

提供以下REST接口：

| 方法 | 路径 | 功能 |
|------|------|------|
| POST | `/api/generate` | 提交文档生成任务（异步，返回task_id） |
| GET | `/api/status/{task_id}` | 查询任务状态和进度 |
| GET | `/api/download/{task_id}` | 下载生成的Word文档 |
| GET | `/api/doc-types` | 获取支持的文档类型列表 |
| POST | `/api/upload` | 上传附件文档 |
| GET | `/api/extract-status/{file_id}` | 查询附件提取状态 |
| GET | `/api/debug/env` | 调试：检查环境变量 |
| GET | `/api/debug/test-api` | 调试：测试MiniMax API |

### 4.3 文档生成引擎（app/services/minimax.py，976行）

这是系统的**核心模块**，实现了完整的分章节生成流水线：

**生成流程（`_generate_by_chapters`）：**
1. 根据文档类型获取章节结构（`DOC_CHAPTERS`，预定义了16种文档类型的章节划分）
2. 对每个章节依次执行：
   - **RAG检索**：从ChromaDB向量库检索相关参考文档（混合检索：向量+BM25，相似度阈值0.3）
   - **跨Collection检索**：额外检索uploads collection中的用户上传附件
   - **Web搜索（Agent SDK优先）**：使用Claude Agent SDK进行智能搜索，失败时回退到Playwright网页抓取
   - **构建增强Prompt**：将RAG结果、Web搜索结果、附件内容注入Prompt
   - **调用MiniMax API**：使用DeepSeek-V4-Pro模型生成章节内容（temperature=0.7，max_tokens=12000）
3. 汇总所有章节内容，返回完整Markdown格式文档

**关键特性：**
- 3次重试机制 + 指数退避（2s/4s）
- 连接超时30s，读取超时180s
- 附件内容智能注入：第一章全文注入（≤3000字），后续章节关键词匹配注入（≤1500字）
- API请求失败时降级为占位符文档

**16种内置Prompt模板：**每种文档类型都有专门的角色设定和质量要求（例如：风险管理报告使用"资深医疗器械风险管理专家，持有ISO 14971内审员资质"）。

### 4.4 RAG检索引擎（app/services/rag/）

#### 4.4.1 向量存储（vector_store.py，484行）

- 基于ChromaDB的持久化向量数据库
- **多Collection查询**：从`medical_device_kb_v2`等collection检索
- **混合检索**：语义向量检索（权重0.6~0.7）+ BM25关键词检索（权重0.3~0.4）
- **智能过滤**：doc_type过滤 + 相似度阈值 + 自动降级（过滤无结果时取消过滤）
- **BM25优化**：大collection（>5万条）自动跳过BM25，避免内存过载

#### 4.4.2 嵌入模型（embedder.py，157行）

- 使用火山方舟`doubao-embedding-vision-250615`模型
- 输出1024维向量
- 3次重试+指数退避
- 逐条调用避免限流（间隔0.1s）

#### 4.4.3 文档摄入（ingest.py，614行）

- 支持格式：.docx / .pdf / .doc / .txt
- **智能分块**：chunk_size=1500，chunk_overlap=200
- **文档类型识别**：基于目录名和文件名关键词自动分类
- **去重**：基于文件内容的MD5 hash生成唯一chunk_id
- **限制**：每文档最多80个chunk

### 4.5 外部搜索服务

#### 4.5.1 Agent SDK搜索（agent_search.py，233行）

- 基于Claude Agent SDK的智能搜索
- Claude自主决策搜索策略（使用WebSearch和WebFetch工具）
- 构建专业研究提示词（指定文档类型、章节、产品信息）
- 超时120秒，结果综合为结构化研究报告
- **优先于Playwright使用**

#### 4.5.2 Playwright Web搜索（web_search.py，635行）

- Headless Chromium浏览器自动化
- 搜索引擎：Bing（中文结果更好），DuckDuckGo作为备用
- 支持深度网页内容抓取（提取main/article/section等标签内容）
- 支持文件下载（.pdf/.doc/.docx/.txt/.xls/.xlsx）
- 内置SOP专用搜索查询和章节查询映射表
- 下载的文件可自动添加到知识库

### 4.6 模板服务（app/services/template.py，238行）

- Word模板加载和填充
- Markdown到Word格式转换：
  - `#` → 文档标题
  - `##` → 一级标题（Heading 1）
  - `###` → 二级标题（Heading 2）
  - `**粗体**` → 粗体段落
  - `- 列表` → 项目符号列表
  - Markdown表格 → Word表格（Table Grid样式，自动识别表头）
  - `---` → 分隔线
  - 代码块 → 引用样式

### 4.7 附件服务（app/services/attachment_service.py，237行）

- 文件格式验证（.docx/.pdf/.txt，最大10MB）
- 后台线程异步提取文本
- MD5去重检查
- 可选的向量库持久化（写入uploads collection）
- 临时文件自动清理

### 4.8 提示词工程（app/services/prompt_engineer.py，228行）

- **系统角色定义**：15年+经验的医疗器械注册文档专家
- **文档类型特定要求**：每种文档类型都有专属的质量检查清单
- **5维质量自检**：内容完整性、专业性、详细程度、语言规范、格式规范

### 4.9 文档生成协调器（app/services/generator.py，115行）

协调整个生成流程的总控模块：
1. 参数验证
2. 附件内容解析（file_ids → 文本）
3. 在线程池中执行AI生成（`asyncio.to_thread`，避免阻塞事件循环）
4. 模板加载和填充
5. 返回Word文件字节流

## 7. 关键设计决策

### 7.1 异步任务模式
- 文档生成采用"提交-轮询-下载"的异步模式，避免长时间HTTP连接超时
- 使用`threading.Thread`后台线程 + `asyncio.new_event_loop()`执行生成
- 附件提取同样采用后台线程处理

### 7.2 多层搜索策略
- **RAG（知识库）** → **Agent SDK搜索** → **Playwright网页搜索** 三层递进
- Agent SDK具有自主搜索策略能力，优先使用；Playwright作为可靠性回退
- 搜索方式在每个章节级别记录并展示给用户

### 7.3 分章节生成
- 将文档拆分为多个章节独立生成，每章有专属的检索查询和Prompt
- 避免单次生成token超限，同时提高每章的针对性和质量
- 章节间间隔0.5s避免API限流

### 7.4 延迟初始化
- RAG组件、Web搜索组件、Agent搜索组件均采用延迟初始化
- 避免启动时的耗时操作，提高服务响应速度
- 不可用时静默降级，不影响核心生成功能

## 10. 项目演进历程

本项目经历了多个迭代阶段：

1. **MVP阶段**：基础的文档生成功能，4种文档类型，单次API调用生成
2. **RAG增强阶段**：引入ChromaDB向量库、文档摄入、语义检索
3. **分章节生成阶段**：从单次生成升级为分章节生成，支持16种文档类型
4. **Web搜索增强阶段**：集成Playwright网页搜索，补充外部法规信息
5. **Agent SDK集成阶段**：引入Claude Agent SDK智能搜索，替代硬编码爬虫
6. **附件功能阶段**：支持上传参考文档，文本提取和向量库入库
7. **知识库扩建阶段**：建设贴敷式胰岛素泵知识库（25分类778文档）
